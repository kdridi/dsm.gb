# Architecture Decision Records (ADR)

## Qu'est-ce qu'un ADR ?
Un ADR documente une décision architecturale ou de gouvernance importante pour le projet.

## Index des ADR

| ID | Titre | Statut |
|----|-------|--------|
| [0001](0001-organisation-scripts.md) | Organisation des scripts | Accepté |
| [0002](0002-conventions-scripts.md) | Conventions de logging et gestion d'erreurs | Accepté |

## Quand créer un ADR ?
- Nouvelle convention ou règle de projet
- Choix technique structurant
- Organisation des fichiers/dossiers
- Tout ce qui doit être respecté systématiquement

## Format
Chaque ADR suit cette structure :
1. **Statut** : Proposé / Accepté / Déprécié
2. **Contexte** : Pourquoi cette décision ?
3. **Décision** : Quoi et comment ?
4. **Conséquences** : Impacts positifs/négatifs

## Nommage
`NNNN-titre-court.md` (ex: `0001-organisation-scripts.md`)
